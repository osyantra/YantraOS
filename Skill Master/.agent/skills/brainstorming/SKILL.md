# Brainstorming & Ideation

Structured methodology for generating, evaluating, and refining ideas. Prevents aimless thinking by applying proven ideation frameworks.

## When to Use This Skill

- The user asks to "brainstorm", "ideate", or "explore ideas"
- The user is stuck and needs creative alternatives
- Early-stage thinking before a project plan exists
- Evaluating multiple approaches to a problem
- Generating feature ideas, names, concepts, or strategies

## Core Workflow

Follow the **Diverge → Organize → Converge → Refine** cycle:

### 1. Diverge (Generate)

Goal: **Maximize quantity.** No judgement, no filtering.

- Ask the user to define the **problem statement** or **opportunity space**
- Apply one or more frameworks from the resources:
  👉 **[`resources/frameworks.md`](resources/frameworks.md)**
- Generate as many ideas as possible
- Capture every idea, even "bad" ones — they spark better ones

Rules during divergence:
- **No criticism** — defer judgement entirely
- **Encourage wild ideas** — they unlock unconventional solutions
- **Build on others** — "Yes, and..." not "No, but..."
- **Aim for volume** — 20+ ideas before filtering

### 2. Organize (Cluster)

Goal: **Find patterns.** Group related ideas into themes.

- Identify natural clusters (3-7 themes)
- Label each cluster with a descriptive name
- Note which clusters have the most energy/ideas
- Identify gaps — are there categories with no ideas?

### 3. Converge (Evaluate)

Goal: **Select the best.** Apply structured evaluation.

Use the **ICE Score** for quick ranking:

| Criterion | Score 1-5 | Description |
| :--- | :--- | :--- |
| **Impact** | How much value does this create? | 1 = minimal, 5 = transformative |
| **Confidence** | How sure are we it will work? | 1 = guess, 5 = proven |
| **Ease** | How easy is it to implement? | 1 = months of work, 5 = hours |

**ICE Score = Impact × Confidence × Ease** (max 125)

Alternatively, use the **2×2 Matrix**:
- X-axis: Effort (Low → High)
- Y-axis: Impact (Low → High)
- Prioritize: High Impact + Low Effort (quick wins)

### 4. Refine (Detail)

Goal: **Flesh out winners.** Turn top ideas into actionable concepts.

For each selected idea, define:
- **One-liner:** What is it in one sentence?
- **Problem it solves:** Why does this matter?
- **How it works:** High-level description
- **Key risks:** What could go wrong?
- **Next step:** What's the first concrete action?

## Framework Quick Reference

| Framework | Best For | How It Works |
| :--- | :--- | :--- |
| SCAMPER | Improving existing things | Substitute, Combine, Adapt, Modify, Put to other use, Eliminate, Reverse |
| How Might We | Reframing problems | Turn pain points into "How might we...?" questions |
| Reverse Brainstorm | Unblocking stuck thinking | Ask "How could we make this worse?" then invert |
| Mind Map | Exploring a broad topic | Central idea → branches → sub-branches |
| Six Thinking Hats | Balanced group evaluation | Facts, Emotions, Caution, Benefits, Creativity, Process |
| Worst Possible Idea | Overcoming creative blocks | Generate terrible ideas deliberately, then flip them |

See detailed framework instructions:
👉 **[`resources/frameworks.md`](resources/frameworks.md)**

## Anti-Patterns to Avoid

| Anti-Pattern | Problem | Solution |
| :--- | :--- | :--- |
| Premature Judgement | Kills idea flow | Separate diverge and converge phases |
| Anchoring | First idea dominates | Generate 20+ before evaluating any |
| Groupthink | Everyone agrees too fast | Use silent ideation before discussion |
| Scope Creep | Session loses focus | Define the problem statement first |
| Analysis Paralysis | Can't pick a winner | Use ICE scoring for objective ranking |

## Best Practices

1. **Define the problem first** — A good question generates better ideas than a vague prompt
2. **Quantity before quality** — Aim for 20+ ideas before any filtering
3. **Use frameworks** — Structure beats freestyle for consistent results
4. **Timebox phases** — 10 min diverge, 5 min organize, 10 min converge, 5 min refine
5. **Write everything down** — Unrecorded ideas are lost ideas
6. **Build on ideas** — "What if we combined X and Y?" unlocks novel solutions
7. **Sleep on it** — If time allows, revisit ideas after a break for fresh perspective
8. **One session, one problem** — Don't mix multiple problem spaces
