# Brainstorming Frameworks

Detailed instructions for each ideation framework. Pick the one that best fits the problem type.

---

## SCAMPER

**Best for:** Improving existing products, features, or processes.

Apply each lens to the existing thing you want to improve:

| Lens | Prompt | Example |
| :--- | :--- | :--- |
| **Substitute** | What can be replaced? | Replace password login with biometric |
| **Combine** | What can be merged? | Combine search and navigation into one bar |
| **Adapt** | What can be borrowed from elsewhere? | Adapt Uber's rating system for freelancers |
| **Modify** | What can be enlarged, reduced, or changed? | Make the onboarding 3 steps instead of 7 |
| **Put to other use** | What else could this be used for? | Use error logs as a customer insight source |
| **Eliminate** | What can be removed? | Remove the mandatory account creation step |
| **Reverse** | What if we did the opposite? | Let customers set their own price |

### How to Run

1. Write the existing product/feature/process at the top
2. Go through each of the 7 lenses
3. Generate 2-3 ideas per lens minimum
4. Mark favorites with ⭐

## How Might We (HMW)

**Best for:** Reframing problems into opportunity spaces.

### How to Run

1. Start with a pain point or problem statement
2. Rewrite it as "How might we...?" questions
3. Each HMW should be broad enough for multiple solutions, narrow enough to be actionable

### Reframing Techniques

| Original Problem | HMW Reframe |
| :--- | :--- |
| "Users abandon checkout" | "HMW make checkout feel effortless?" |
| "Customers don't read docs" | "HMW make help available where they already are?" |
| "Onboarding is confusing" | "HMW make first-time use feel like second-time use?" |

### Quality Check

A good HMW question:
- ✅ Inspires multiple different solutions
- ✅ Focuses on the user's need, not a specific solution
- ❌ Is NOT so broad it's meaningless ("HMW make everything better?")
- ❌ Is NOT a solution disguised as a question ("HMW add a chatbot?")

## Reverse Brainstorm

**Best for:** Getting unstuck when ideas aren't flowing.

### How to Run

1. **Invert the problem** — Ask: "How could we make this *worse*?"
2. **Generate terrible ideas** — List ways to guarantee failure
3. **Flip each one** — Turn each bad idea into its opposite
4. **Evaluate the flips** — Some will be genuinely innovative

### Example

Problem: "How to improve user retention?"

| Worst Idea | Flipped Idea |
| :--- | :--- |
| Send 50 emails a day | Smart, minimal notifications based on behavior |
| Hide the unsubscribe button | Make leaving easy; focus on earning their stay |
| Make the app crash randomly | Invest in bulletproof reliability as a feature |
| Remove all helpful features | Surface the most-used feature on every visit |

## Mind Map

**Best for:** Exploring a broad topic and discovering unexpected connections.

### How to Run

1. Write the **central topic** in the middle
2. Branch out with **main categories** (3-7 branches)
3. Add **sub-branches** for each category
4. Add **connections** between branches where related
5. Look for **unexpected clusters** — these are often the best ideas

### Format

```
                    ┌── Sub-idea 1
         ┌── Branch A ──┤
         │              └── Sub-idea 2
Central ──┤
         │              ┌── Sub-idea 3
         └── Branch B ──┤
                        └── Sub-idea 4
```

## Six Thinking Hats

**Best for:** Evaluating ideas from multiple perspectives to avoid bias.

### The Hats

| Hat | Color | Focus | Key Question |
| :--- | :--- | :--- | :--- |
| 📊 | White | Facts & Data | What do we know? What data exists? |
| ❤️ | Red | Emotions & Gut | How does this feel? What's the intuition? |
| ⚠️ | Black | Caution & Risks | What could go wrong? What are the dangers? |
| ☀️ | Yellow | Benefits & Value | What's the best case? Why will this succeed? |
| 💚 | Green | Creativity | What alternatives exist? What's unconventional? |
| 🔵 | Blue | Process | Are we on track? What's the next step? |

### How to Run

1. Start with 🔵 Blue — define the topic and process
2. Move to 📊 White — gather known facts
3. Then ❤️ Red — capture initial gut feelings
4. Apply ☀️ Yellow — explore all the benefits
5. Apply ⚠️ Black — identify risks and concerns
6. Apply 💚 Green — generate creative alternatives
7. Close with 🔵 Blue — decide next steps

## Worst Possible Idea

**Best for:** Breaking creative blocks and reducing pressure to be "right."

### How to Run

1. **Announce the rule** — We are *only* generating terrible ideas
2. **Go wild** — The worse the idea, the better
3. **Have fun** — Laughter lowers inhibition and sparks creativity
4. **Find the gems** — Ask "What's actually interesting about this bad idea?"
5. **Extract insights** — The kernel of a bad idea often contains an unconventional truth

This works because removing the pressure to be clever frees up creative thinking.
